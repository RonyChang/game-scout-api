# Documentation Index

This directory is the versioned source of truth for product and technical documentation.

## Structure

- `product/` project vision and goals
- `architecture/` architecture and system diagrams
- `api/` API contracts and behavior
- `decisions/` architecture decision records (ADRs)
- `runbooks/` workflows and operations
- `changelog/` chronological change log

## Documentation Rule

For every technical change, you MUST do one of the following:

1. Update one or more relevant docs files.
2. Add an explicit "no documentation impact" note in the change workflow artifact.
